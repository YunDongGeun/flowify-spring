# 1. 개요

---

## 1.1 시스템의 목표

본 Flowify 시스템은 비전문가가 AI 자동화 파이프라인을 코드 없이 구축할 수 있도록 설계된 **노드 기반 워크플로우 플랫폼**의 Spring Boot 메인 백엔드를 제공하는 것을 목표로 한다.

기존 자동화 도구(n8n, Zapier, Make)의 높은 진입 장벽을 극복하여, **[입력 → AI 처리 → 출력]** 이라는 직관적인 3단계 구조로 비전문가도 즉시 자동화를 구축할 수 있는 환경을 제공한다.

### Spring Boot 백엔드 목표

- **인증/인가 관리**: Google OAuth 2.0 SSO 기반 단일 로그인 및 JWT 토큰 기반 세션 관리를 통해 안전하고 편리한 사용자 인증 제공
- **워크플로우 관리**: 노드 기반 워크플로우의 생성, 조회, 수정, 삭제 및 소유권/공유 관리를 통한 워크플로우 라이프사이클 관리
- **실행 위임 및 모니터링**: 워크플로우 실행을 FastAPI AI 서비스에 위임하고, 실행 이력 및 로그를 프론트엔드에 제공
- **외부 서비스 토큰 관리**: OAuth 토큰의 암호화 저장, 자동 갱신, 안전한 전달을 통한 외부 서비스 연동 지원
- **템플릿 관리**: 시스템 기본 제공 및 사용자 생성 워크플로우 템플릿 관리
- **데이터 무결성 및 보안**: 사용자별 데이터 격리, 토큰 암호화, 내부 통신 인증을 통한 보안 보장

## 1.2 소프트웨어의 주요 기능

### 1.2.1 회원 및 인증 관리 기능

Google OAuth 2.0 SSO를 통한 단일 로그인을 지원하며, JWT 토큰 기반의 인증/인가 체계를 제공한다.

- **Google SSO 로그인**: Google OAuth 2.0 Authorization Code Grant 플로우를 통한 사용자 인증 및 자동 회원가입
- **JWT 인증**: Access Token(30분) + Refresh Token(7일) 발급 및 검증
- **인가 처리**: Spring Security 필터 체인을 통한 엔드포인트별 접근 제어

### 1.2.2 사용자 관리 기능

로그인된 사용자의 정보 조회, 수정, 탈퇴 기능을 제공한다.

- **사용자 정보 조회/수정**: 현재 로그인 사용자의 프로필 관리
- **회원 탈퇴**: 사용자의 모든 관련 데이터(워크플로우, OAuth 토큰, 실행 이력) 일괄 삭제

### 1.2.3 워크플로우 CRUD 기능

노드 기반 워크플로우의 전체 라이프사이클을 관리한다.

- **워크플로우 CRUD**: 워크플로우의 생성, 목록 조회(페이지네이션), 상세 조회, 수정, 삭제
- **소유권 관리**: user_id 기반 소유권 검증 및 공유 설정
- **유효성 검증**: 순환 참조, 필수 노드 누락, 연결 끊김 등 워크플로우 구조 검증 (EXR-05)

### 1.2.4 워크플로우 실행 위임 기능

워크플로우 실행을 FastAPI AI 서비스에 위임하고 실행 결과를 관리한다.

- **실행 요청 위임**: 워크플로우 정의 + 복호화된 서비스 토큰을 FastAPI에 전달
- **실행 이력 관리**: 실행 상태, 노드별 로그, 스냅샷 기반 롤백 지원 (EXR-06)
- **노드별 데이터 미리보기**: 실행 완료 후 각 노드의 입력/출력 데이터 제공

### 1.2.5 템플릿 관리 기능

워크플로우 템플릿을 통해 빠른 자동화 구축을 지원한다.

- **시스템 템플릿**: 학습 노트 자동 생성, 회의록 요약 및 공유, 뉴스 수집 및 정리, 구글 시트 리포트 PDF 등 기본 제공 템플릿
- **사용자 템플릿**: 사용자가 본인의 워크플로우를 템플릿으로 저장
- **템플릿 인스턴스화**: 템플릿 기반 워크플로우 생성

### 1.2.6 OAuth 토큰 관리 기능

외부 서비스(Google, Slack, Notion) 연동을 위한 OAuth 토큰을 안전하게 관리한다.

- **토큰 암호화 저장**: AES-256-GCM 알고리즘으로 액세스/리프레시 토큰 암호화
- **토큰 자동 갱신**: Lazy Refresh 방식으로 만료 임박 토큰 자동 갱신 (EXR-02)
- **FastAPI 전달**: 워크플로우 실행 시 복호화된 토큰을 안전하게 전달

## 1.3 설계상 제약사항

1. **서버 분리**: Spring Boot 메인 백엔드와 FastAPI AI 서비스는 설계, 구현, 시험 단계부터 분리되어 별도의 서버에서 각각 작동하도록 구성된다.

2. **프론트엔드 단일 접점**: 프론트엔드(React)는 Spring Boot에만 요청하며, FastAPI는 외부에 직접 노출하지 않는다.

3. **내부 통신 인증**: Spring Boot ↔ FastAPI 간 통신은 `X-Internal-Token` 공유 시크릿 헤더를 통해 인증하며, `X-User-ID` 헤더로 사용자 컨텍스트를 전파한다.

4. **JWT 서명 알고리즘**: 단일 서버 환경에 적합한 HS256(대칭키) 알고리즘을 사용한다.

5. **OAuth 토큰 보안**: 사용자별 OAuth 토큰은 AES-256-GCM으로 암호화하여 user_id 단위로 격리 저장하며, API 응답에 절대 노출하지 않는다.

6. **워크플로우 소유권**: 모든 워크플로우의 소유권은 user_id 단위로 엄격하게 관리하며, 서비스 레이어에서 반드시 검증한다.

7. **Stateless 인증**: REST API 특성에 맞게 CSRF를 비활성화하고, Session 관리를 STATELESS로 설정한다.

8. **데이터베이스**: MongoDB(Document DB)를 주 데이터베이스로 사용하며, Spring Data MongoDB를 통해 접근한다.

9. **기술 스택 고정**:
   - Spring Boot 3.4.3 / Java 21
   - Gradle (Groovy DSL)
   - 패키지명: `org.github.flowify`
   - Lombok 적극 활용
   - Docker 기반 배포

10. **외부 API 재시도 정책**: 외부 서비스 API 오류 시 최대 3회 Exponential Backoff 재시도, LLM API 오류 시 최대 2회 재시도를 적용한다 (EXR-01, EXR-03).
