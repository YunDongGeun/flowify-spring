### 3.3 예외 요구사항

---

#### EXR-01: 외부 서비스 API 연결 오류

| 항목 | 내용 |
|------|------|
| **요구사항 식별자** | EXR-01 |
| **요구사항명** | 외부 서비스 API 연결 오류 |
| **구분** | 예외 요구사항 |
| **발생 조건** | Google, Slack, Notion 등 외부 서비스 API 호출 시 네트워크 오류, 타임아웃, HTTP 5xx 응답이 발생한 경우 |
| **시스템 처리** | 최대 3회 자동 재시도(Exponential Backoff) 후 실패 시 해당 노드를 FAILED 상태로 전환하고, 오류 원인과 재시도 방법을 사용자에게 안내한다. 실행 로그에 오류 상세 정보를 기록한다. |

#### EXR-02: OAuth 인증 오류

| 항목 | 내용 |
|------|------|

| **시스템 처리** | Refresh Token을 이용한 자동 갱신을 시도한다. 갱신 실패 시 워크플로우 실행을 중단하고 사용자에게 재인증을 요청하는 안내 메시지를 표시한다. |

#### EXR-03: LLM API 오류

| 항목 | 내용 |
|------|------|
| **요구사항 식별자** | EXR-03 |
| **요구사항명** | LLM API 오류 |

| **발생 조건** | LLM(EXAONE) API 호출 시 서버 오류, Rate Limit 초과, 타임아웃 등이 발생한 경우 |
| **시스템 처리** | Rate Limit 초과 시 대기 후 자동 재시도한다. 서버 오류 시 최대 2회 재시도 후 실패 시 해당 AI 노드를 FAILED 상태로 전환하고, 오류 원인과 수동 재실행 방법을 안내한다. |

#### EXR-04: LLM 기반 워크플로우 자동 생성 실패

| 항목 | 내용 |
|------|------|
| **요구사항 식별자** | EXR-04 |
| **요구사항명** | LLM 기반 워크플로우 자동 생성 실패 |
| **구분** | 예외 요구사항 |
| **발생 조건** | 사용자의 자연어 입력이 모호하거나 LLM 응답이 유효한 워크플로우 구조로 변환할 수 없는 경우 |
| **시스템 처리** | LLM 응답의 유효성을 검증하고, 유효하지 않은 경우 사용자에게 입력 내용을 구체화하도록 추가 질문을 제시한다. 반복 실패 시 템플릿 기반 생성을 대안으로 안내한다. |
