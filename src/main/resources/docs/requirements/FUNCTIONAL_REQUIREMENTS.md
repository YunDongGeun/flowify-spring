# 기능적 요구사항

---

## 1. 개요

### 1.1 시스템 개요

Flowify는 비전문가가 AI 자동화 파이프라인을 코드 없이 구축할 수 있도록 설계된 노드 기반 워크플로우 플랫폼이다.

현재 대부분의 일반 사용자는 ChatGPT 등의 생성형 AI를 일회성 질의응답 도구로만 활용하고 있으며, AI 결과물을 Notion, Google Drive 등 외부 도구로 옮기는 과정에서 반복적인 수작업이 발생하고 있다. 기존 자동화 도구인 n8n, Zapier, Make는 이 문제를 해결할 수 있는 기능을 제공하지만, 노드 개념 이해, 데이터 매핑 설정, OAuth 인증 등 개발자 중심의 설계로 인해 비전문가에게는 높은 진입 장벽이 존재한다.

Flowify는 유사한 목적의 서비스들을 도메인 카테고리 단위로 묶어 노드로 제공한다. 사용자는 커뮤니케이션, 저장소, 스프레드시트, 웹 수집, 캘린더 5개의 도메인 노드를 클릭으로 선택하고, 각 노드 내부에서 서비스 선택, 트리거, 조건 분기, 반복, 알림 등의 기능을 단계별로 설정한다. 데이터 수집 방식, 서비스 간 데이터 규격 통합, LLM 연동 등 기술적으로 복잡한 처리는 시스템 내부에서 자동으로 결정되어 사용자에게 노출되지 않는다. 자연어 입력을 통한 AI 기반 워크플로우 자동 생성과 템플릿 기반 생성을 함께 제공하여 비전문가도 즉시 자동화를 구축할 수 있는 환경을 목표로 한다.

### 1.2 시스템 목표

#### 기능 목표

커뮤니케이션(Gmail, Slack), 저장소(Google Drive, Notion), 스프레드시트(Google Sheets), 웹 수집(쿠팡, 원티드, 네이버 스마트스토어, 네이버 뉴스, 구글 뉴스), 캘린더(Google Calendar) 5개 도메인 노드를 제공한다. 각 노드는 트리거, 필터링, 반복, 조건 분기, AI 처리, 알림 등의 기능을 내부적으로 선택적으로 포함한다. LLM 기반 플로우 자동 생성과 템플릿 기반 워크플로우 생성을 통해 다양한 수준의 사용자가 자동화를 즉시 구축할 수 있도록 한다.

#### 사용성 목표

노드 선택부터 설정 완료까지 대부분의 과정을 마우스 클릭만으로 완료할 수 있도록 설계한다. 자연어 프롬프트 입력은 AI 처리 노드에서만 허용하며, 나머지 설정은 단계별 선택지 제시 방식으로 진행된다. LLM 응답 대기 중 단계별 실시간 미리보기를 제공하여 사용자 경험을 보완한다.

#### 성능 및 안정성 목표

LLM 처리 노드는 비동기 방식으로 실행되어 전체 파이프라인 지연을 방지한다. Loop 노드에는 최대 반복 횟수와 타임아웃을 적용하여 무한 루프를 차단한다. 노드 실행 전후 상태를 스냅샷으로 저장하여 실패 시 롤백 및 재실행이 가능하도록 보장한다.

#### 보안 목표

Google OAuth SSO를 통한 단일 로그인을 지원하며, 서비스별 OAuth 토큰은 암호화하여 user_id 단위로 격리 저장한다. 모든 워크플로우의 소유권은 user_id 단위로 엄격하게 관리한다.

#### 확장성 목표

Strategy 패턴과 Factory 패턴을 적용하여 새로운 노드 추가 시 기존 워크플로우 엔진 코드를 수정하지 않아도 되는 구조를 유지한다. 개방-폐쇄 원칙(OCP)을 준수하여 향후 도메인 노드 및 서비스 연동 확장에 유연하게 대응할 수 있도록 설계한다.

#### UC-U01: Google SSO 로그인

| 항목 | 내용 |
|------|------|
| **유스케이스 식별자** | UC-U01 |
| **유스케이스 명** | Google SSO 로그인 |
| **개요** | 사용자가 Google 계정을 이용하여 시스템에 로그인한다. |
| **주 액터** | 사용자 |
| **선행 조건** | - 사용자는 Google 계정을 보유하고 있어야 한다.<br>- 시스템은 Google OAuth 로그인 기능을 지원해야 한다. |
| **후행 조건** | 사용자는 시스템에 로그인된 상태가 된다. |

**기본 흐름:**

| 단계 | 액터 | 시스템 |
|------|------|--------|
| 1 | 사용자가 메인 화면에서 Google 로그인 버튼을 선택한다. | |
| 2 | | 시스템은 Google 인증 페이지로 사용자를 이동시킨다. |
| 3 | 사용자가 Google 계정으로 로그인하고 권한 제공에 동의한다. | |
| 4 | | 시스템은 Google로부터 사용자 인증 정보를 수신한다. |
| 5 | | 인증 성공 시 인증 여부가 승인된다. |

**예외 상황 시나리오:**

| 식별자 | 조건 | 시스템 처리 |
|--------|------|------------|
| E1 | 로그인 취소 | 사용자가 로그인 도중 취소할 경우 로그인 절차를 종료하고 메인 화면으로 돌아간다. |
| E2 | 인증서버 통신 오류 | Google 인증 서버와 통신이 실패할 경우 시스템은 오류 원인과 재시도 안내 메시지를 표시한다. |

| 항목 | 내용 |
|------|------|
| **유스케이스 식별자** | UC-U02 |
| **유스케이스 명** | 서비스별 OAuth 인증 |
| **개요** | 사용자가 Gmail, Google Drive, Slack 등 기타 외부 서비스와 연동하기 위해 OAuth 인증을 수행한다. |
| **주 액터** | 사용자 |

| **후행 조건** | 선택한 외부 서비스가 시스템과 연동된다. |

**기본 흐름:**

| 단계 | 액터 | 시스템 |

| 3 | | 시스템은 해당 서비스의 OAuth 인증 화면으로 이동시킨다. |
| 4 | 사용자가 권한 제공에 동의한다. | |
| 5 | | 시스템은 인증 토큰을 발급받아 저장한다. |
| 6 | | 시스템은 연동 완료 상태를 사용자에게 표시한다. |

**예외 상황 시나리오:**

| 식별자 | 조건 | 시스템 처리 |
