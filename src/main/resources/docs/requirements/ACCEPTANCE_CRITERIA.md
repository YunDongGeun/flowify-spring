| FT-W03 | 도착 노드 설정 | UC-W01-B | 카테고리 → 서비스 → 동작 유형을 선택한다. | 도착 노드가 확정되고, 세부 설정은 이 시점에서 진행되지 않는다. |
| FT-W04 | 중간 과정 직접 설정 (3단계 흐름) | UC-W01-D | "+" 버튼 클릭 → 1차 처리 방식 선택 → 2차 데이터 타입별 선택 → 3차 후속 설정 완료. | 양측 패널(좌: 입력 데이터+선택지, 우: 출력 데이터)이 표시되고, 사용자 선택에 따라 시스템이 노드 타입을 결정한다. |
| FT-W05 | LLM 기반 채팅형 생성 | UC-W02 | "AI 생성" 선택 후 채팅 인터페이스에서 AI 질문에 응답한다. | AI가 반복 질문을 통해 정보를 수집하고, "충분" 판단 시 워크플로우가 자동 생성된다. 직접 설정과 동일한 방식으로 수정 가능하다. |
| FT-W06 | 템플릿 기반 플로우 생성 | UC-W03 | 템플릿 리스트에서 템플릿 선택 → 상세보기 → "가져오기" 클릭. | 워크플로우가 생성되어 리스트에 추가되고, 편집 화면이 열린다. 모든 노드가 설정된 상태로 표시된다. |
| FT-W07 | 미연동 서비스 포함 템플릿 생성 | UC-W03 | 미인증 서비스가 포함된 템플릿을 "가져오기"한다. | 워크플로우가 생성되되, 해당 노드에 인증 경고가 표시된다. (워크플로우 미생성이 아님) |
| FT-W08 | 노드 삭제 캐스케이드 | UC-W01-D | 중간 노드를 삭제한다. | 해당 노드 이후의 모든 후속 노드가 함께 삭제된다. |
| FT-W09 | 이전 단계 변경 호환성 경고 | UC-W01-D | 이미 설정된 이전 노드의 선택을 변경한다. | 이후 노드는 삭제되지 않고 유지되되, 호환성 경고가 표시된다. |
| FT-W10 | 데이터 타입 호환성 검증 | UC-W01-D | 중간 과정의 출력 데이터가 도착 노드 입력과 호환되지 않는 워크플로우를 구성한다. | 호환성 경고가 사용자에게 표시된다. |
| FT-W11 | 조건 분기 독립 경로 | UC-W01-D | 분류 기준을 선택하여 조건 분기 노드를 생성한다. | 분기된 각 경로마다 독립적인 "+" 버튼이 표시되고, 각 경로별로 독립적인 설정이 가능하다. |
| FT-W12 | 도착 노드 세부 설정 (지연) | UC-W01-D | 중간 과정 완료 후 도착 노드를 선택한다. | 들어오는 데이터에 맞는 세부 설정 화면이 열린다. |

### 4.2 성능 시험

| 용어 | 설명 |
|------|------|
| 워크플로우 (Workflow) | 복수의 노드가 순서대로 연결되어 데이터를 처리하는 자동화 파이프라인 |
| 노드 (Node) | 워크플로우를 구성하는 개별 처리 단위. 입력, AI 처리, 로직, 출력 등의 역할을 수행 |
| 캔버스 (Canvas) | 노드를 시각적으로 배치하고 연결하는 워크플로우 편집 영역 |
| 트리거 (Trigger) | 워크플로우 실행을 시작하는 조건. 시간 기반 또는 이벤트 기반으로 설정 |
| LLM (Large Language Model) | 대규모 언어 모델. 본 시스템에서는 OpenAI GPT-4o를 사용하여 자연어 처리 수행 |
| GPT-4o | OpenAI에서 개발한 멀티모달 대규모 언어 모델 |
| LangChain | LLM 기반 애플리케이션 개발을 위한 프레임워크. 프롬프트 관리, 체이닝, 에이전트 기능 제공 |
| OAuth 2.0 | 외부 서비스 접근 권한을 안전하게 위임하기 위한 인증 프로토콜 |
| SSO (Single Sign-On) | 하나의 인증으로 여러 서비스에 접근할 수 있는 통합 인증 방식 |
| RAG (Retrieval-Augmented Generation) | 벡터 검색 기반 지식 베이스를 활용하여 LLM 응답 품질을 향상시키는 기법 |
| Chroma | 오픈소스 임베딩 데이터베이스. 벡터 저장 및 유사도 검색 지원 |
| Strategy 패턴 | 알고리즘을 캡슐화하여 동적으로 교체할 수 있게 하는 디자인 패턴. 노드 실행 로직에 적용 |
| Factory 패턴 | 객체 생성 로직을 별도 클래스에 위임하는 디자인 패턴. 노드 인스턴스 생성에 적용 |
| State 패턴 | 객체의 상태에 따라 행동을 변경하는 디자인 패턴. 워크플로우 상태 관리에 적용 |
| Adapter 패턴 | 호환되지 않는 인터페이스를 변환하여 연결하는 디자인 패턴. 이기종 데이터 규격 통합에 적용 |
| 스냅샷 (Snapshot) | 노드 실행 전후의 상태를 저장한 데이터. 오류 발생 시 롤백에 활용 |
| 롤백 (Rollback) | 오류 발생 시 이전 상태로 복원하는 작업 |
| 비동기 처리 (Async) | 작업 완료를 기다리지 않고 다음 작업을 수행하는 처리 방식. LLM 호출 등에 적용 |
| 웹 크롤링 (Web Crawling) | 웹사이트에서 자동으로 데이터를 수집하는 기술 |
| DTO (Data Transfer Object) | 계층 간 데이터 전달을 위한 객체. 이기종 서비스 간 공통 데이터 포맷으로 활용 |
